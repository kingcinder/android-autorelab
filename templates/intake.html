Choose one option

How do you want to begin?

All three options lead into the same guided intake review before any run is created.

Option 1

Acquire from physical target device

Use this when you have the target hardware in front of you and want to capture the initial lab context before analysis begins.

  • Best for first-time acquisitions
  • Captures device label, connection hint, and custody notes

Option 2

Load saved project

Use this when work already exists and you want to resume it without losing track of the original provenance.

  • Best for returning to an existing case
  • Keeps any missing provenance explicit

Option 3

Load reference file or reference file set

Use this when you already have firmware packages, extracted images, or prior evidence and want the system to normalize them first.

  • Best for firmware bundles, images, and evidence archives
  • Keeps imported material workflow-neutral until you choose a lane
