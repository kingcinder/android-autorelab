# Target Intake Notes

The startup intake flow is shared and neutral until the operator explicitly binds the session to The Agency or The Legion.

Supported intake wording:

- Acquire from physical target device
- Load saved project
- Load reference file or reference file set

Reference file examples:

- downloaded firmware files from SamFW
- vendor firmware packages
- extracted images
- prior evidence bundles
- normalized metadata bundles

Intake records:

- what was provided
- what was inferred
- what remains unknown
- provenance and acquisition notes

Workflow binding happens only after intake context is established. After binding, all files remain under `runs/<workflow>/...`.
